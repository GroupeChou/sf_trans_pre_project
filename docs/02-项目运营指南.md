# 中转场地在线预测智能体平台 — 项目运营指南

> 版本：v1.0 | 日期：2026-06-03 | 面向角色：平台运营团队、系统管理员、业务培训师

---

## 一、平台概述

**转运场地在线预测智能体平台**是一个面向全国 1000+ 物流中转场地的多租户 SaaS 预测平台。核心能力：

- 业务人员用自然语言描述预测需求，系统自动选择最优预测方法组合
- 内置 15+ 种标准化预测 Skill（同环比、客户摸底、天气调整、倒货管理等）
- 支持业务人员自主拖拽组装预测方法（Skills Composer）
- 支持自然语言即时生成临时预测方法（Prompt-to-Skill）
- 全链路可审计、可解释、可追溯

## 三、日常运营流程

### 3.1 场地运营 — 每日预测流程

```
17:00  打开钉钉/Web
       ↓
17:00  输入预测需求（自然语言）
       "预测明天华新白班到件量，得物有直播，可能有台风"
       ↓
17:01  收到预测卡片
       · 主值: 132,000 件
       · 置信区间: [115,000, 149,000]
       · 共识度: 85% 🟢
       · 方法: B1+M4+F1
       ↓
17:02  看一眼，判断是否合理
       ├── 合理 → 点 [确认采纳] → 自动推送给 IE
       └── 不合理 → 点 [人工调整] → 输入调整值 + 原因
```

### 3.2 区域经理 — 每日巡检流程

```
08:30  打开 Web 管理台 → 区域风险总览
       ↓
       查看：
       · 哪些场地预测共识度低（DI > 0.30）？
       · 哪些场地数据缺失严重？
       · 哪些场地连续多天人工调整率高？
       ↓
09:00  重点关注的场地：
       · 点进去看详情 → 看 Agent 主张分歧 → 判断是否需要协调资源
```

### 3.3 预测管理员 — 每周维护

- **周一**：检查各 Skill 的 MAPE 趋势、降级率、采纳率
- **周三**：审核方法论社区中新提交的方案（认证/驳回/建议修改）
- **周五**：检查场地画像是否需要更新（新场地、场地类型变更等）

### 3.4 数据科学家 — Skill 生命周期管理

```
开发 → 提交审核 → 回放测试 → 管理员 Review → 发布 → 监控效果 → 迭代/弃用
```

## 四、关键场景运营指南

### 4.1 大促期间（618/双11/年货节）

**提前 1 周**：
- 预测管理员检查大促模板（T2_大促峰值）的参数是否需要更新
- 确认大客户摸底数据通道（kadm）可用
- 为高频场地预配置促销 Skills 组合

**大促当天**：
- 场地运营使用"618 预测"触发词自动加载大促模板
- 密切关注客户上报件量与实际件量的偏差
- 如有重大偏差（> 20%），立即人工调整并标注原因

**大促后**：
- 数据科学家分析大促期间各 Skill 的表现
- 更新 Platt Scaling 校准系数

### 4.2 恶劣天气应对

**收到天气预警后**：
1. 场地运营在预测文本中包含天气关键词（"台风"、"暴雨"、"暴雪"）
2. 系统自动启用 F1 极端事件 Skill + 调整衰减系数
3. 输出含风险标记的预测（⚠️ 外部事件影响）
4. 建议输出区间预测而非点预测

**注意**：恶劣天气下模型可靠性降低，建议：
- 关注宽置信区间
- 必要时人工覆盖
- 恶劣天气结束后重新预测

### 4.3 新场地上线

1. 平台管理员在场地画像中创建新场地记录
2. 标记 `is_cold_start = true`
3. 系统自动使用 S3 同城类比 + V1 基期校验
4. 前 30 天所有预测标注"新场地，数据积累中"
5. 30 天后场地画像自动更新，切换为正常模式

### 4.4 兜底场地管理

兜底场地（如金山 021WD）的特点是件量被倒货主导，预测不确定性大：
- 始终叠加 G2 倒货管理 Skill + 安全余量（15%）
- Web 看板上显示倒货率趋势、来源场地状态
- 预测卡片显式标注"包含规划外倒货影响"

## 五、系统监控

### 5.1 关键指标

| 指标 | 计算方式 | 告警阈值 | 检查频率 |
|------|---------|:---:|:---:|
| 系统可用率 | 成功返回/总请求 | < 95% 🔴 | 实时 |
| 预测 P95 延迟 | 95% 请求的响应时间 | > 60s 🟡 | 每小时 |
| 降级率 | 降级模式/总请求 | > 10% 🟡 | 每天 |
| 人工调整率 | 调整次数/总预测 | > 50% 🟡 | 每周 |
| Token 消耗 | 日均 Token × 30 | 超预算 20% 🔴 | 每天 |
| LLM API 可用率 | 成功/总 LLM 调用 | < 99% 🔴 | 实时 |

### 5.2 告警处理流程

```
告警触发 → 值班人员确认
  ├── 系统故障 → 检查依赖服务状态（DB/Redis/LLM API）
  │   ├── LLM 不可用 → 自动切换纯规则模式
  │   ├── DB 不可用 → 仅 F6 兜底
  │   └── 记录→通知→修复→复盘
  │
  └── 业务异常 → 检查特定场地/场景
      ├── 某场地 MAPE 突增 → 检查数据质量/事件标签
      └── 通知预测管理员跟进
```

## 六、数据治理

### 6.1 数据备份

- 预测记录：每日增量备份，保留 365 天
- 审计日志：实时写入，保留 365 天
- Skill 权重：每日快照，保留 90 天
- 场地画像：每次修改版本化

### 6.2 数据隐私

- 租户间数据严格隔离（独立 Schema 或 tenant_id 过滤）
- LLM 调用不传输原始敏感业务数据（仅传输聚合统计值）
- 人工调整记录包含操作者身份和原因，不可篡改

## 七、培训体系

### 7.1 场地运营培训（30 分钟）

1. **基础预测**（10 分钟）：怎么用自然语言发起预测、看结果卡片
2. **场景模板**（10 分钟）：618 模板、台风模板怎么一键调用
3. **人工调整**（10 分钟）：什么情况下需要调整、怎么调整、为什么要写原因

### 7.2 预测管理员培训（2 小时）

1. Skills 市场管理：注册/启停/调权
2. 场地画像配置：方法偏好/默认参数设置
3. 分析台使用：如何看 MAPE 趋势、如何识别问题场地
4. 方法论社区运营：方案认证/精华贴管理

### 7.3 常见问题

**Q：系统选的 Skills 和我觉得应该用的不一样怎么办？**
A：点击 [我要自己调] → 进入 Skills Composer → 拖拽选你想要的 Skills → 对比效果 → 保存为你的模板

**Q：预测值明显偏了怎么办？**
A：直接人工调整，写入调整原因（≥5 字）。系统不会改你的值，但会记录这次调整用于未来学习。

**Q：数据缺失了怎么办？**
A：系统会自动降级。数据恢复后会自动重新预测并推送更新通知。你也可以点击 [手动输入数据] 临时补充。

## 八、持续改进

### 8.1 月度运营复盘

- **MAPE 趋势**：各场地/各场景/各 Skill 的准确率变化
- **采纳率**：哪些预测被直接采纳？哪些总被人工调整？
- **Token 成本**：预算执行率、优化空间
- **用户满意度**：通过钉钉机器人推送简易问卷

### 8.2 季度竞品对标

- 跟踪顺丰超脑、京东京慧的功能更新
- 确认差异化定位是否依然有效
- 调整产品路线图优先级

### 8.3 反馈渠道

- 钉钉群：日常使用问题、需求建议
- Web 管理台"帮助与反馈"入口
- 方法论社区：方法讨论、参数调优经验分享
