# 中转场地在线预测智能体平台 — 阶段性需求开发计划

> 版本：v1.0 | 日期：2026-06-03 | 基于 PRD v3.2 + 技术选型开发指南

---

## 一、总览

本计划将平台建设划分为 **4 个阶段、12+ 个月**，从 MVP 验证到全网 1000+ 场地覆盖。核心原则：

- **快速验证**：Phase 0 用 2 个月在 8 个试点场地跑通端到端闭环
- **渐进增强**：每个阶段在上阶段基础上叠加智能能力，不推翻重来
- **业务驱动**：优先交付业务人员最痛的场景（日常预测、大客户调整、天气应对）
- **成本可控**：Token 分级管控、LLM 只做语义理解不做数值计算

---

## 二、Phase 0：MVP 验证（0-8 周）

### 目标
在 8 个试点场地实现"钉钉输入自然语言 → 10 秒返回预测卡片"的端到端闭环。

### 交付范围

| 模块 | 内容 | 不包括 |
|------|------|--------|
| 预测方法 | 6 个核心 Skill（B1/D1/F1/G2/H1/F6） | 高级统计方法 |
| 交互入口 | 钉钉/Web 单轮预测 + 人工调整 | 多轮对话 |
| 智能路由 | 意图解析 + 三维评分选 Skill | LLM Debate |
| 结果返回 | 预测卡片（主值+置信区间+方法贡献） | 全链路详情穿透 |
| 数据通道 | NL2SQL 手动封装 | 全自动 API |
| 管理后台 | 管理员手动配置 | Skills 市场 UI |

### 关键里程碑

| 周 | 里程碑 | 验收标准 |
|----|--------|---------|
| W1-2 | 项目脚手架 + 核心框架搭建 | Evidence Blackboard + ForecastClaim Schema + Skill 基类 |
| W3-4 | 6 个核心 Skill 开发完成 | B1/D1/F1/G2/H1/F6 单元测试通过 |
| W5-6 | 编排引擎联调 | 12 节点 Pipeline 跑通，端到端测试通过 |
| W7 | 8 场地试点部署 | 日常预测 MAPE ≤ 10% |
| W8 | MVP 验收 | 7 项功能验收全部通过 |

### 资源需求

| 角色 | 人数 | 投入 |
|------|:---:|------|
| 后端工程师 | 2 | 全职 |
| 算法/数据工程师 | 1 | 全职 |
| 产品经理 | 1 | 50% |
| 场地运营（业务配合） | 2 | 20% |

### MVP 验收清单

| # | 验收项 | 条件 |
|---|--------|------|
| 1 | 自然语言识别 | 20 条真实话术，场地+日期识别率 ≥ 95% |
| 2 | Skill 自动选择 | 每次预测自动选择 2-5 个 Skill |
| 3 | Skill 并行执行 | 独立 Skill 并行，总耗时 ≤ 串行之和的 50% |
| 4 | 预测结果返回 | 发起→返回卡片 ≤ 3 分钟 |
| 5 | 人工调整闭环 | 支持调整值+原因+标签，生成审计记录 |
| 6 | 降级兜底 | 任一 Skill 失败不中断，F6 兜底值始终输出 |
| 7 | 准确率达标 | MAPE ≤ 人工基线的 110% |

## 三、Phase 1：全量 Skill + 智能增强（9-16 周）

### 目标
补齐全部 15 套方案 Skill，引入 LLM Debate 智能化协商，开放 Skills Composer 和 Prompt-to-Skill。

### 新增交付

| 模块 | 内容 |
|------|------|
| 剩余 Skills | S1-S5 统计方法、D1-D6 维度拆分、V1-V5 校验 |
| Debate | LLM 有界辩论（1-3 轮），分歧自动检测 |
| Skills Composer | 可视化拖拽组装 + 参数微调 + 方案对比 |
| Prompt-to-Skill | 自然语言描述 → 临时预测方法（限 3 次使用） |
| 方法论社区 | 方案保存/分享/评价（基础版） |
| satis 模型接口 | 生产模型作为 S0 Skill 注册到市场 |
| 四层兜底 | 数据降级 + Skill 降级，全面容灾 |
| Web 管理台 | Skills 市场管理 + 场地画像配置 |

### 关键里程碑

| 周 | 里程碑 |
|----|--------|
| W9-10 | 补齐剩余 Skills，全部 12 个方案上线 |
| W11-12 | Debate 协商机制 + 质量门禁 |
| W13-14 | Skills Composer 完整版 + Prompt-to-Skill MVP |
| W15 | satis 接口适配 + 四层兜底基础实现 |
| W16 | Phase 1 验收 + 方法论社区上线 |

## 四、Phase 2：规模化 + 分析治理（17-32 周）

### 目标
推广至 50 场地，建立完整的分析、审计、治理体系。

### 新增交付

| 模块 | 内容 |
|------|------|
| 多场地推广 | 50 场地覆盖，数据自动化覆盖 80% |
| Agent 表现分析台 | MAPE/采纳率/降级率 多维度看板 |
| 决策轨迹审计 | 全链路回放、问题追溯 |
| 场景模板体系 | 预配置 Skill 组合，业务一键调用 |
| Agent 记忆体系 | 短期+中期 Agent 记忆，跨任务学习 |
| Nexus 双分辨率 | Macro(城市趋势) + Micro(场地波动) 双视角 |
| DAAO 难度评分 | 场地波动数据积累 30 天后，DAAO 模型上线 |
| 四层兜底完整版 | LLM 降级 + 业务兜底 |
| 方法论社区完整版 | 排行 + 衍生 + 专家认证 |

---

## 五、Phase 3：全网覆盖 + 自进化（33-52 周）

### 目标
覆盖 1000+ 场地，实现 Agent 自进化和跨场地知识迁移。

### 新增交付

| 模块 | 内容 |
|------|------|
| 全网覆盖 | 1000+ 场地，零手动录入 |
| AgentLoop | 预测→反馈→权重/参数/校准自动调整闭环 |
| Peer Review | Worker 间对等质询，去中心化 Debate |
| 跨场地知识迁移 | 方法改进自动推荐到相似场地 |
| DyLAN 动态组装 | MAB 收敛后启用查询级 Skill 子集动态选择 |
| 容灾自动切换 | 无人工干预的故障恢复 |
| HiClaw 生态对接 | Matrix 协议 + 企业级 IM 集成 |
| 竞品自动化对标 | 季度竞品分析报告自动生成 |

## 七、风险与缓解

| 风险 | 等级 | 缓解策略 |
|------|:---:|---------|
| NL2SQL 服务延期 | 🔴 高 | Phase 0 可降级为手动 SQL 封装 |
| satis/OE API SLA 不足 | 🟡 中 | 增加重试+缓存，F6 兜底始终在线 |
| 业务人员学习成本高 | 🟡 中 | 渐进式：模板→调参→自由组装 |
| Token 成本超预期 | 🟡 中 | 三级 Token 预算管控，Prompt Caching |
| Prompt-to-Skill 代码质量不可控 | 🟡 中 | 沙箱隔离 + 3 次使用限制 + 必审核 |
